# ⚔ V Rising — Сайт игрового сервера

Веб-сайт для игрового сервера **V Rising**: мониторинг серверов в реальном времени, новости с комментариями и реакциями, лидерборд игроков, вайпы, FAQ и полноценная панель администратора. Разворачивается одной командой на чистом Debian 13, обновляется командой `vrising`.

---

## Стек технологий

| Слой | Технология |
|------|-----------|
| Backend | Python 3.12, FastAPI (async) |
| База данных | SQLite (aiosqlite), SQLAlchemy 2.0 (async ORM) |
| Авторизация | JWT (python-jose) + bcrypt (passlib) |
| Frontend | HTML5, Tailwind CSS (CDN), Vanilla JS, Canvas 2D (графики) |
| ИИ-чат | Anthropic API ("Управляющий замком", опционально) |
| Reverse Proxy | Nginx (+ опционально HTTPS через Let's Encrypt) |
| Контейнеризация | Docker + Docker Compose |
| ОС сервера | Debian 13 (Trixie) |

## Функциональность

### Главная страница (`index.html`)
- Виджет мониторинга обоих игровых серверов в реальном времени (онлайн/офлайн, число игроков, карта, версия), с кнопкой подключения через `steam://rungameid/...`
- Обратный отсчёт/история вайпов
- Лента новостей: пагинация, теги, закреплённые посты, реакции, комментарии, модальное окно с полным текстом
- Опциональный ИИ-чат «Управляющий замком» (Anthropic API)
- Боковая навигация на все разделы сайта

### Сервера (`servers.html`)
- Подробный мониторинг каждого из двух серверов: статус, список игроков онлайн
- Интерактивные графики истории онлайна (Canvas, сглаженные кривые) с переключением периода
- Часовая тепловая карта активности (heatmap), привязанная к выбранному периоду
- История и статистика вайпов

### Игроки (`leaderboard.html`)
- Полный лидерборд по суммарному времени в игре, отдельно по каждому серверу
- Переключение периода: всё время / месяц / неделя
- Поиск игрока по имени (с debounce)
- Подсветка и кнопка «📍 Найти себя» — переходит на страницу со своей строкой в рейтинге
- Индикатор «онлайн сейчас» у игроков, подключённых в данный момент
- Длительность последней сессии
- Аватары игроков (подтягиваются по совпадению игрового имени с именем учётной записи на сайте), пьедестал почёта для топ-3

### Кланы (`clans.html`)
- Раздел в разработке (заглушка): данные о кланах будут поступать напрямую из игрового плагина сервера, без ручного ведения списка
- Backend API кланов (создание, вступление, выход, удаление, передача лидерства) уже реализован и используется в админ-панели — готов к подключению, когда появится интеграция с плагином

### Карта (`map.html`)
- Информационный обзор регионов карты Вардоран (Farbane Woods, Dunley Farmlands, Silverlight Hills, Cursed Forest, Hallowed Mountains, Gloomrot, Brighthaven, подземелья) с уровнем опасности каждого региона

### Баны (`bans.html`)
- Раздел в разработке (заглушка)

### FAQ (`faq.html`)
- Аккордеон с ответами на частые вопросы: подключение через Steam, вайпы, правила PvE, регистрация и привязка игрового имени, восстановление пароля, кланы, онлайн на серверах, сообщение о нарушениях

### Авторизация и профиль
- Регистрация и вход по логину/паролю, JWT-токены, пароли хэшируются через bcrypt
- Запоминание входа (remember me), восстановление пароля по email (forgot/reset password)
- Личный профиль (`profile.html`) и публичные профили игроков (`user.html`) — статистика, клан, аватар
- Две роли: `user` (игрок) и `admin` (администратор)
- Мастер первоначальной настройки сайта (`setup.html`) при первом запуске

### Панель администратора (`admin.html`)
- Дашборд со сводной статистикой
- Управление новостями: создание, редактирование, удаление, черновики, закрепление, теги, загрузка изображений
- Модерация комментариев
- Управление пользователями: роли, блокировка/разблокировка, удаление
- Управление кланами (просмотр, удаление)
- Управление лидербордом: удаление записей игроков
- Управление вайпами серверов
- Настройки сайта: название, фон, IP/порт серверов, установка SSL-сертификата, обновление сайта с GitHub из интерфейса
- Файловый менеджер загруженных файлов
- Журнал аудита действий администраторов
- Управление запросами на восстановление пароля

## Быстрый старт — Debian 13

### Автоматическая установка (рекомендуется)

```bash
git clone https://github.com/RJ-Bond/vrising-server-site.git
cd vrising-server-site
sudo bash install.sh
```

Скрипт автоматически:
1. Обновит пакеты системы
2. Установит Docker и Docker Compose (официальный репозиторий)
3. Настроит брандмауэр UFW (порты 80, 443, SSH)
4. Соберёт и запустит контейнеры
5. Создаст администратора и базу данных
6. Зарегистрирует системные команды `vrising` (установка/обновление) и `vrising-https` (выпуск SSL)

После завершения в терминале появятся адрес сайта и данные для входа.

### Обновление

На сервере, где сайт уже установлен (`/opt/vrising-site`), для обновления до последней версии из репозитория достаточно выполнить:

```bash
sudo vrising
```

Команда подтянет изменения из GitHub, пересоберёт и перезапустит контейнеры.

---

### Ручная установка

**1. Установите Docker**

```bash
curl -fsSL https://get.docker.com | sh
```

**2. Клонируйте репозиторий**

```bash
git clone https://github.com/RJ-Bond/vrising-server-site.git
cd vrising-server-site
```

**3. Создайте файл `.env`**

```bash
cp .env.example .env
```

Отредактируйте `.env`:

```env
SECRET_KEY=замените_на_случайную_строку_32_символа
DATABASE_URL=sqlite+aiosqlite:////data/vrising.db
VRISING_SERVER_IP=127.0.0.1
VRISING_SERVER_PORT=27016
ANTHROPIC_API_KEY=опционально_для_чата_управляющий_замком
```

**4. Запустите проект**

```bash
docker compose up -d --build
```

Сайт будет доступен на порту `80`.

## API

Документация Swagger доступна после запуска по адресу:  
`http://<IP-сервера>/api/docs`

| Раздел | Примеры роутов | Доступ |
|--------|------------------|--------|
| Авторизация | `POST /api/auth/register`, `/login`, `/logout`, `/auth/me`, `/auth/change-password`, `/auth/forgot-password`, `/auth/reset-password/{token}`, `/auth/avatar` | Публичный / User |
| Первоначальная настройка | `GET /api/setup/status`, `POST /api/setup/complete` | Публичный (до первой настройки) |
| Мониторинг серверов | `GET /api/monitor/status[2]`, `/monitor/history[2]`, `/monitor/snapshots`, `/monitor/stats` | Публичный |
| Новости | `GET /api/news`, `/news/{slug}`, `/news/tags`, реакции и комментарии | Публичный / User |
| Вайпы | `GET /api/wipes`, `POST/DELETE /api/admin/wipes` | Публичный / Admin |
| Лидерборд | `GET /api/leaderboard`, `DELETE /api/admin/leaderboard/{id}` | Публичный / Admin |
| Кланы | `GET/POST/PUT /api/clans`, `/clans/{id}/join`, `/clans/leave`, `DELETE /api/admin/clans/{id}` | Публичный / User / Admin |
| Профили | `GET /api/users/{username}` | Публичный |
| ИИ-чат | `POST /api/chat` | User |
| Админ: контент | `GET/POST/PUT/DELETE /api/admin/news`, `/admin/comments`, `/admin/upload`, `/admin/uploads` | Admin |
| Админ: пользователи | `GET /api/admin/users`, `PUT /api/admin/users/{id}/role`, `/toggle-active`, `DELETE` | Admin |
| Админ: настройки | `GET/PUT /api/admin/settings`, `/admin/settings/import`, `/admin/ssl/install`, `/admin/update` | Admin |
| Админ: служебное | `GET /api/admin/stats`, `/admin/audit-log`, `/admin/password-resets` | Admin |

```bash
# Просмотр логов

# Остановка
docker compose down

docker compose up -d --build

```
